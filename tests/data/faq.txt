MChat 常见问题解答 (FAQ)

1. MChat 是什么？
MChat 是一个多租户垂直 RAG（检索增强生成）平台，结合了流式对话引擎、知识库管理、技能插件和工作流编排功能。

2. 如何创建知识库？
登录管理后台，在「知识库」菜单中点击「新建知识库」，填写名称和描述，选择 Embedding 提供商和模型，然后点击保存。创建完成后可以上传文档到该知识库。

3. 知识库支持哪些检索方式？
支持三种检索方式：
- 向量检索（vector）：使用 Embedding 模型进行语义匹配
- 关键词检索（keyword）：使用 BM25 算法进行精确匹配
- 混合检索（hybrid）：结合以上两种方式，通过 RRF（倒数排名融合）算法合并结果

4. 如何配置本地 Ollama 模型？
首先确保已安装 Ollama，然后拉取所需模型：
ollama pull nomic-embed-text  # 用于 Embedding
ollama pull qwen2.5           # 用于对话
在管理后台的「模型工作台」中，新增一个 AI 配置，提供商选择 ollama，模型选择 qwen2.5。

5. 知识库中的文档如何更新？
进入知识库详情页，你可以：
- 点击「添加文档」上传新文件
- 点击「重新索引」更新已有文档的向量
- 点击「删除」移除不需要的文档

6. 什么是 Rerank（重排序）？
Rerank 是在初次检索后对结果进行二次精排的技术。MChat 支持三种重排序方式：
- lexical：基于关键词重叠的轻量级重排序
- Cohere：使用 Cohere API 进行语义重排序
- BGE：使用 BGE 模型进行本地重排序
- 关闭（none）：不进行重排序

7. 知识库检索的准确率如何提升？
建议：
- 选择合适的 chunk_size（推荐 500-1000）
- 启用混合检索模式
- 根据内容特点选择分块策略（markdown 文档用 markdown 策略，技术文档用 fixed 策略）
- 开启 Rerank 功能
- 定期更新知识库内容

8. 支持哪些文件格式？
支持以下格式的文档上传：
- 文本文件（.txt）
- Markdown（.md）
- Word 文档（.docx）
- PDF 文件（.pdf）
- HTML 文件（.html）

9. 如何测试知识库的检索效果？
进入知识库详情页，在「搜索测试」输入框中输入查询语句，即可查看检索结果和相关性分数。

10. 知识库是否有容量限制？
目前没有文档数量限制，但建议单个知识库的文档总数不超过 10000 个，以确保检索性能。每个文件的大小限制为 50MB。
